字 垣 · 书签指纹预览

独立测试页 —— 浏览器内直接调参 + 比对，验证指纹四要素
本页指纹算法逐字移植自 src/game/fingerprint.ts（buildFingerprint / rotateMatrix / cellHash / synthesizeFingerprintInput），与本页卡片样式移植自 src/components/BookmarkCard.tsx。 四要素：① 卡片形状（镂空排布）· ② 用时占比（外圈顺时针点亮）· ③ 角度（道具徽章按角度顺时针落位 TL→TR→BR→BL，内层亦整体旋转）· ④ 是否用道具（徽章颜色：纯解=绿 / 道具=琥珀）。改了 TS 侧请同步本页 <script>。
镂空(正解位)/纯解徽章 定位符/计时框 道具徽章 卡身实体 背景(空)

A · 四要素对照（同一题，各变一项）

基准 = 千里之行始于足下 / 中等 / 固定不对称镂空 / 种子 12345。每张只改一个要素，看指纹如何变。

B · 角度要素（同一镂空，0/90/180/270）

道具徽章按角度顺时针落位 TL(0°)→TR(90°)→BR(180°)→BL(270°)，角度一目了然；内层镂空亦随之旋转。

C · 用时要素（外圈顺时针点亮）

用时越多，外圈计时框（金）越满，沙漏意象。0% 全空 → 100% 整圈点亮（跳过四角定位符）。

D · 道具要素（右下徽章）

纯解 → 右下徽章绿；用过道具 → 右下徽章琥珀。其余完全一致。

E · 卡片形状要素（不同名言 → 不同镂空）

镂空按名言字数 + 种子决定性散布（合成兜底路径，与老存档一致）。字数不同 → 镂空数/排布不同。
